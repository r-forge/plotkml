kml {plotKML}R Documentation

Wrapper function to plot any Spatial* or Raster* object in a KML file.

Description

This function allows the user to plot any Spatial* object (from the sp package) or Raster* object (from the raster package) to be plotted in a KML file. Various aesthetics, i.e. ways to represent information, are provided (colour, transparency, size, shape, etc.). Their availability depends on the class of the object to plot.

Usage

kml(obj, folder.name, file.name, overwrite = TRUE, kmz = FALSE, ...)

Arguments

obj object inheriting from the Spatial* or the Raster* classes
folder.name folder name text
file.name output file name (*.kml)
overwrite logical; overwrites the existing file
kmz logical; specifies whether to compress the kml file
... Arguments interpreted in the context of kml() (see details below)

Value

A KML file located as per the file option.

Author(s)

Pierre Roudier (pierre.roudier@landcare.nz), Tomislav Hengl (tom.hengl@wur.nl), Dylan Beaudette (debeaudette@ucdavis.edu)

See Also

kml_open, kml_close, kml_layer

Examples

# Plotting a SpatialPointsDataFrame object
data(eberg)
coordinates(eberg) <- ~X+Y
proj4string(eberg) <- CRS("+init=epsg:31467")
## Not run: 
# Simple plot
kml(eberg, file = "eberg-0.kml")
# Plot using aesthetics
kml(eberg, colour = SAND, size = CLAY, alpha = 0.75, file = "eberg-1.kml")
## End(Not run)
# Plotting a SpatialPixelsDataFrame or a Raster* object
data(eberg_grid)
## Not run: 
kml(eberg_grid, colour = DEM, file = "eberg_grid-0.kml")
## End(Not run)
