plotKML.env {plotKML}R Documentation

plotKML specific environmental variables / paths

Description

The function automatically looks for FWTools, ImageMagick, Python, SAGA GIS, in the Windows Registry Hive, the Program Files directory or the usr/bin installation (Unix).

Usage

plotKML.env(colour_scale_numeric = "", colour_scale_factor = "", ref_CRS = "",
    kml_xsd = "", gpx_xsd = "", convert = "", gdalwarp = "", gdal_translate = "",
    python = "", home_url = "", googleAPIkey = "", show.env = TRUE)

Arguments

colour_scale_numeric default colour scheme for numeric variables
colour_scale_factor default colour scheme for factor variables
NAflag the default missing value flag (usually "-99999")
ref_CRS the referent CRS proj4string ("+proj=longlat +datum=WGS84")
kml_xsd the default KML scheme URL
gpx_xsd the default GPX scheme URL
convert a path to ImageMagick convert program
gdalwarp a path to FWTools gdalwarp program
gdal_translate a path to FWTools gdalwarp program
python a path to Python program
home_url the default location of all icons and auxiliary files
googleAPIkey (optional) Google API key required for some operations

Details

The function will try to locate external software tools under either Windows or Unix platform and then save the results to plotKML.opts environment. plotKML-package does not look automatically for software paths (unless you specify this manually in your "Rprofile.site").
The external software tools are not required by default and most of operations in plotKML-package can be run without using them. FWTools, SAGA GIS and Python are highly recommended, however, for processing large data sets.

Warning

Under Linux OS make sure you add path to executable program by using e.g.:
export PATH=INSTALL_DIRECTORY/FWTools-2.*.*/bin_safe:$PATH

Note

To further customize the plotKML options, consider putting library(plotKML); plotKML.env(..., show.env = FALSE) in your "/etc/Rprofile.site".

Examples

## Not run: 
pts <- paths()
pts
# look for paths:
plotKML.env(silent = FALSE)
gdalwarp <- get("gdalwarp", envir = plotKML.opts)
# if missing you need to install it!
system(paste(gdalwarp, "--help-general")) 
gdal.formats <- system(paste(gdalwarp, "--formats"), intern = TRUE)
## End(Not run)
plotKML.env(show.env = FALSE)
get("home_url", envir = plotKML.opts)
