gstatModel-class {plotKML}R Documentation

A class for a linear geostatistical model

Description

A class containing fitted parameters of a linear geostatistical model. It contains regression (GLM) and the kriging part (variogram).

Slots

regModel:

object of class "glm"; fitted GLM

sp:

object of class "SpatialPointsDataFrame"; observation locations

vgmModel:

object of class "data.frame"; the fitted gstat variogram model parameters

Methods

summary

signature(obj = "gstatModel"): summarize object by showing the mapping accuracy (cross-validation) and the amount of variation explained by the model

predict

signature(obj = "gstatModel"): makes predictions for a set of given predictionLocations (gridded maps)

Note

"SpatialPredictions" saves results of predictions for a single target variable, which can be of type numeric or factor. Multiple variables can be combined into a list. When using nsim argument with the predict method, the output result will be of type:

RasterBrickSimulations-class

i.e. N number of equiprobable realizations. To generate an object of type:

SpatialPredictions-class

use nsim = 0.

Author(s)

Tomislav Hengl

See Also

SpatialPredictions-class, RasterBrickSimulations-class, gstat::gstat-class

Examples

 
# load observations:
data(meuse)
coordinates(meuse) <- ~x+y
proj4string(meuse) <- CRS("+init=epsg:28992")
# load grids:
data(meuse.grid)
coordinates(meuse.grid) <- ~x+y
gridded(meuse.grid) <- TRUE
proj4string(meuse.grid) <- CRS("+init=epsg:28992")
# fit a model:
omm <- fit.gstatModel(observations = meuse, formulaString = om~dist, 
covariates = meuse.grid)
show(omm@regModel)
# produce SpatialPredictions:
om.rk <- predict(omm, predictionLocations = meuse.grid)
# overview of the prediction process:
show(om.rk)
## Not run: # plot the whole project and open in Google Earth:
plotKML(om.rk)

## End(Not run)
